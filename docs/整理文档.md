# 📁 项目文档整理说明

## 文档分类

为了更好地组织项目，我们将文档分为以下几类：

### 1. 根目录文档（用户必读）
保留在项目根目录，方便用户快速了解项目：

- ✅ `README.md` - 项目介绍和快速开始
- ✅ `LICENSE` - 开源许可证
- ✅ `CONTRIBUTING.md` - 贡献指南
- ✅ `CHANGELOG.md` - 版本更新日志
- ✅ `.gitignore` - Git 忽略文件
- ✅ `.env.example` - 环境变量示例

### 2. 构建相关文档
移动到 `docs/build/` 目录：

- `如何构建APK.md`
- `BUILD_INSTRUCTIONS.md`
- `构建APK步骤.md`
- `开始构建.md`
- `APK构建完成.md`

### 3. 开发文档
移动到 `docs/` 目录：

- `ARCHITECTURE.md` - 架构文档
- 其他技术文档

### 4. 任务和检查点文档
移动到 `docs/tasks/` 目录：

- `CHECKPOINT_*.md`
- `TASK_*.md`
- `PROJECT_STATUS.md`

### 5. 示例文件
移动到 `docs/examples/` 目录：

- `*.example.tsx`

## 整理步骤

### 自动整理（推荐）

运行整理脚本：
```bash
cd cloudflare-analytics
./scripts/organize-docs.sh
```

### 手动整理

1. **创建文档目录**
   ```bash
   mkdir -p docs/build
   mkdir -p docs/tasks
   mkdir -p docs/examples
   mkdir -p docs/screenshots
   ```

2. **移动构建文档**
   ```bash
   mv 如何构建APK.md docs/build/
   mv BUILD_INSTRUCTIONS.md docs/build/
   mv 构建APK步骤.md docs/build/
   mv 开始构建.md docs/build/
   mv APK构建完成.md docs/build/
   ```

3. **移动任务文档**
   ```bash
   mv CHECKPOINT_*.md docs/tasks/
   mv TASK_*.md docs/tasks/
   mv PROJECT_STATUS.md docs/tasks/
   mv SETUP_NOTES.md docs/tasks/
   ```

4. **移动示例文件**
   ```bash
   mv src/screens/*.example.tsx docs/examples/
   ```

5. **更新 README 中的链接**
   - 更新文档链接指向新位置
   - 例如: `./如何构建APK.md` → `./docs/build/如何构建APK.md`

## 清理不需要的文件

可以删除或移动到归档目录：

```bash
# 创建归档目录
mkdir -p docs/archive

# 移动旧文档
mv *.md docs/archive/ 2>/dev/null || true
```

## Git 提交

整理完成后提交更改：

```bash
git add .
git commit -m "docs: reorganize project documentation"
```

## 注意事项

1. **保留重要文档**: 不要删除 README、LICENSE 等重要文件
2. **更新链接**: 移动文件后更新所有引用链接
3. **测试构建**: 确保移动文件后构建脚本仍然有效
4. **备份**: 整理前先备份重要文件

## 维护建议

1. **新文档**: 按类别放入对应目录
2. **定期清理**: 删除过时文档
3. **保持一致**: 遵循命名规范
4. **更新索引**: 在 README 中维护文档索引

## 文档命名规范

- **英文文档**: `UPPERCASE.md` (例如: `README.md`)
- **中文文档**: `中文名称.md` (例如: `如何构建APK.md`)
- **技术文档**: `kebab-case.md` (例如: `api-reference.md`)
- **任务文档**: `TASK_XX_description.md`
- **检查点**: `CHECKPOINT_XX_description.md`
